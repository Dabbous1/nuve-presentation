Slide 13

Security & Compliance Architecture

Multi-layered security for financial services

Authentication & Access

  • JWT + Refresh Token rotation (15 min access tokens)
  • Biometric unlock via React Native Biometrics
  • SMS OTP for sensitive ops (withdrawal, profile change)
  • Device fingerprinting + session binding
  • Express rate limiting on all auth endpoints

Data Protection

  • AES-256 encryption at rest (PostgreSQL TDE + MinIO SSE)
  • TLS 1.3 for all data in transit (Nginx termination)
  • Per-user encrypted KYC document storage in MinIO
  • PII hashing for analytics (no raw PII in logs)
  • Data retention policies per FRA requirements

Regulatory Compliance

  • FRA Non-Banking Financial Services regulations
  • Egyptian Data Protection Law (No. 151/2020)
  • CBE Payment regulations (Phase 2+)
  • PCI-DSS Level 1 (Phase 3 — card issuance only)
  • AML screening via sanctions list API

Operational Security

  • Firebase Crashlytics error monitoring (free)
  • GitHub Actions CI/CD with SAST scanning
  • Infrastructure-as-Code (Ansible + Terraform)
  • Penetration testing before each phase launch
  • SOC 2 Type II target (Phase 2)
